Abstract

This vocabulary describes various small activities I like to log as an extension to ActivityStreams 2.0. I would expect them to present to a reader in aggregate over some time period.

Discussion

Notes about things considered when creating this vocabulary.

  • Things are fairly arbitrarily added as I decide I want to add them to my site. Deliberately short and simple.
  • These activities may not have objects because all they are for is notifications of aggregate things.

If I've missed something PR/issue welcome on github.

Commits

Motivated by wanting a contributions tracker like Github has, but that includes Bitbucket and private repos as well. I made a githook that automatically generates a Commit activity and posts it to my site using SocialPub whenever I make a commit locally.

Commit

A commit activity. Subclass of as:Activity.

Words

Motivated by using 750words and also wanting to track wordcounts for Nanowrimo and my thesis, I wrote a script that parses my past 750words outputs and turns them into Write activities per day (which I then dumped into my triplestore), and a script that takes a markdown file and creates a Write activity and posts it with SocialPub on run, and an extension for dokieli that sends a Write activity with Solid on a button press.

Write

A write activity. Subclass of as:Activity.

Instant Messages

Motivated by how much time I spend on IRC; I wrote a script that converts the results of a SQLite query on my Quassel logs to Send activities (which I can then dump in my triplestore).

Commit

A commit activity. Subclass of as:Activity.

Usage

This vocabulary is used by (and was initially developed for) my website, for storing my logs because I felt like it. Example:

@prefix diet: <http://vocab.amy.gy/logs#>
<http://rhiaro.co.uk/about#me>
                                .

If you start using it for something or publishing your own logs somewhere using this, let me know or PR to add yours here.
